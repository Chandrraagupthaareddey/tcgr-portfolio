# Deployment — Cloudflare Pages + tcgr.in

This guide covers production deployment from a GitHub push to a live `tcgr.in` URL with auto-deploy on every commit.

## Architecture summary

```
git push origin main
      ↓
Cloudflare Pages build
      ↓
build.sh injects API_URL into public/config.js
      ↓
Static deploy of /public to *.pages.dev + tcgr.in
```

---

## Part 1 — Push the repo to GitHub

```bash
# In repo root
git init
git add .
git commit -m "Initial commit — sheet-driven portfolio for tcgr.in"
git branch -M main

# Create the GitHub repo via UI or gh CLI:
gh repo create tcgr-in/portfolio --public --source=. --push
```

> Verify nothing sensitive was committed:
> ```bash
> git ls-files | grep -E "(\.clasp\.json|\.env|secrets)"
> # Should return nothing
> ```

## Part 2 — Add the build script

The build script reads env vars from Cloudflare and writes them into `public/config.js` at build time. Without this, the site has no way to know your Apps Script URL.

Create `build.sh` in the repo root (already included in this repo):

```bash
#!/usr/bin/env bash
set -euo pipefail

# Cloudflare Pages provides these as env vars
API_URL="${API_URL:-}"
SITE_VERSION="${SITE_VERSION:-1.0.0}"
CACHE_TTL_MS="${CACHE_TTL_MS:-21600000}"  # 6h default

cat > public/config.js <<EOF
/* Auto-generated by build.sh — do not edit. */
window.PORTFOLIO_CONFIG = {
  API_URL: "${API_URL}",
  CACHE_TTL_MS: ${CACHE_TTL_MS},
  SITE_VERSION: "${SITE_VERSION}",
  SHOW_FALLBACK_BANNER: false
};
EOF

echo "✓ Built public/config.js with API_URL=[redacted] and version=${SITE_VERSION}"
```

Make it executable:
```bash
chmod +x build.sh
git add build.sh
git commit -m "Add Cloudflare Pages build script"
git push
```

## Part 3 — Connect Cloudflare Pages

1. Go to [dash.cloudflare.com](https://dash.cloudflare.com) → **Workers & Pages → Create → Pages → Connect to Git**
2. Authorize GitHub, select your `portfolio` repo
3. **Set up builds and deployments:**

   | Setting | Value |
   |---|---|
   | Project name | `tcgr` (or whatever you want for the *.pages.dev URL) |
   | Production branch | `main` |
   | Framework preset | None |
   | Build command | `sh build.sh` |
   | Build output directory | `public` |
   | Root directory | (leave blank) |

4. **Environment variables** (Production):

   | Variable | Value |
   |---|---|
   | `API_URL` | Your Apps Script Web App URL (`https://script.google.com/macros/s/.../exec`) |
   | `SITE_VERSION` | `1.0.0` |
   | `CACHE_TTL_MS` | `21600000` |

5. Click **Save and Deploy**

After ~30 seconds, your site is live at `https://tcgr.pages.dev`. Open it — you should see real data from your Sheet.

> **Verifying it worked:** Open browser DevTools → Network tab → look for a request to your Apps Script URL returning JSON. If you see only fallback data, the env var didn't apply — check Pages → Settings → Env vars.

## Part 4 — Custom domain (tcgr.in)

### If your domain DNS is on Cloudflare already

1. Cloudflare Pages project → **Custom domains → Set up a custom domain**
2. Enter `tcgr.in`
3. Cloudflare adds the CNAME automatically. SSL provisions in <60s.
4. Add `www.tcgr.in` too (creates a redirect).

### If your domain is at another registrar

1. In your registrar, add these DNS records:
   ```
   Type: CNAME
   Name: @ (or apex/root)
   Value: tcgr.pages.dev
   ```
2. Or — better — transfer DNS to Cloudflare (free):
   - Cloudflare → **Add site → tcgr.in**
   - Cloudflare gives you 2 nameservers
   - Update nameservers at your registrar
   - Wait for propagation (5min – 48h)

After propagation, `tcgr.in` resolves to your Pages site with auto-SSL.

## Part 5 — Auto-deploy workflow

Now every `git push origin main` triggers:

1. Cloudflare Pages clones the repo
2. Runs `sh build.sh` → injects env vars into `config.js`
3. Deploys `/public` to CDN
4. Live in ~30 seconds

**Preview deployments**: every PR automatically gets a unique `*.pages.dev` URL for review.

## Common operations

| Task | How |
|---|---|
| **Update content** | Edit Sheet → wait 6h, or click 🛠 → Clear Cache |
| **Update code** | `git push` → Cloudflare auto-deploys |
| **Update backend** | `clasp push` → in Apps Script: Deploy → Manage deployments → Edit → New version |
| **Rotate API_URL** | New deployment in Apps Script → update `API_URL` env var in Cloudflare → trigger rebuild |
| **Roll back frontend** | Cloudflare Pages → Deployments → click old deployment → "Rollback to this deployment" |
| **Check live site is healthy** | Open `https://tcgr.in/?nocache=1` and inspect Network tab |

## Troubleshooting

| Symptom | Likely cause | Fix |
|---|---|---|
| Site shows fallback data only | `API_URL` env var missing or wrong | Check Cloudflare Pages → Settings → Environment variables |
| CORS error in console | Apps Script not deployed as Web App, or "Anyone" access not set | Re-deploy with correct settings |
| Edits to Sheet don't appear | Cache hasn't expired (6h default) | 🛠 → Clear Cache in Sheet menu |
| Build fails on Cloudflare | `build.sh` not executable | `chmod +x build.sh && git push` |
| Custom domain shows "not secure" | SSL hasn't provisioned yet | Wait 5–10 minutes; if still failing, remove and re-add the domain |
| Apps Script returns "Authorization required" | Forgot to set "Anyone" access on deploy | Re-deploy → Web app → Who has access: Anyone |

That's it. Edit a cell, see it live within 6 hours. Push code, see it deployed in 30 seconds.
